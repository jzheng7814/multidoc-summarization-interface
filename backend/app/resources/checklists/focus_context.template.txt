Target run title: #RUN_TITLE

This run supports structured extraction and human review over a multi-document corpus centered on one target subject.

Purpose
- Build a checklist that is accurate enough for downstream summary generation and easy for a human reviewer to verify from the cited evidence.
- Optimize for strong, self-contained values and canonical supporting evidence.
- The human reviewer should mostly need to delete, trim, or correct unsupported content, not reopen documents to discover major facts the extraction missed.

Scope discipline
- Extract facts about the target subject itself unless a checklist item explicitly asks for relationships, citations, or external matters.
- External references, quoted background material, and parallel matters are not target-subject facts unless the local text explicitly ties them to the target subject.
- Do not infer key attributes, decisions, or status updates for the target subject from adjacent or cited materials.

Source priority
- Prefer the most authoritative local source for each item.
- Prefer direct primary-source passages over summaries of those passages elsewhere in the corpus.
- When later material clearly updates, narrows, corrects, or supersedes earlier material, the later supported material controls.

Value quality
- Write each value as a concise, self-contained synthesis that remains useful without the evidence text.
- Do not output bare labels, citations, or thin paraphrases when a more informative supported value is possible.
- Prefer fewer, stronger, summary-usable values over many weak fragments.
- When new evidence enriches, narrows, or corrects an existing proposition, consolidate and update instead of appending duplicates.

Evidence quality
- Cite evidence that directly supports every material clause of the value.
- Prefer canonical spans that let a reviewer confirm the value quickly.
- If one span does not support the full value, use multiple spans or split the value into smaller supported values.
- Do not rely on a span that supports only a fragment when the value makes a broader claim.

Pitfalls to avoid
- Do not confuse the target subject with cited examples, background references, or adjacent matters.
- Do not treat quoted or summarized external material as a fact about the target subject unless the local text explicitly applies it.
- Do not encode uncertainty as Not Applicable or N/A unless the record affirmatively establishes non-applicability and the checklist item explicitly calls for that outcome.

Completeness standard
- Before stopping on an important item, check the canonical sources most likely to contain the answer.
- Prefer capturing the major summary-significant developments of the target subject over enumerating minor noise.
- If later material materially changes an earlier understanding, capture that change explicitly.
- When a bucket is relation-heavy or easy to contaminate, leave it empty rather than filling it with unsupported inferences.
